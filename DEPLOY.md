# 🚀 วิธีเอาเว็บขึ้นออนไลน์ (ฟรี)

เป้าหมาย: ให้ลูกค้าเปิดลิงก์จากมือถือเครื่องไหนก็ได้เพื่อสั่ง แล้วหลังบ้านเห็นทันที

มี 2 ขั้นตอนใหญ่: **(1) อัปโค้ดขึ้น GitHub → (2) เชื่อมกับโฮสต์**

---

## ขั้นที่ 1 — อัปโค้ดขึ้น GitHub

1. สมัคร/เข้าสู่ระบบ GitHub ที่ https://github.com
2. กดปุ่ม **New repository** ตั้งชื่อเช่น `cat-food-order` เลือก Public แล้วกด Create
3. อัปโหลดไฟล์ในโฟลเดอร์นี้ทั้งหมด (ยกเว้น `node_modules`) เข้า repo
   - วิธีง่ายสุดถ้าไม่ถนัด command line: ในหน้า repo กด **Add file → Upload files** แล้วลากไฟล์ทั้งหมด (`server.js`, `package.json`, `.gitignore`, `README.md`, `DEPLOY.md` และโฟลเดอร์ `public`) เข้าไป → กด Commit

> เคล็ดลับ: ลากโฟลเดอร์ `public` เข้าไปด้วยเพื่อให้โครงสร้างครบ

---

## ขั้นที่ 2 — เลือกโฮสต์

### ตัวเลือก A: Railway (แนะนำ — ข้อมูลไม่หาย) ⭐

Railway เก็บข้อมูลถาวรได้ด้วย "Volume" เหมาะกับการใช้งานจริง (มีเครดิตทดลองฟรี หลังจากนั้นค่าใช้จ่ายเล็กน้อยตามการใช้)

1. เข้า https://railway.app → Login ด้วย GitHub
2. กด **New Project → Deploy from GitHub repo** → เลือก repo `cat-food-order`
3. Railway จะ build และรันให้อัตโนมัติ (อ่านจาก `npm start`)
4. **สำคัญ — ทำให้ข้อมูลไม่หาย:** เข้าที่ service → แท็บ **Variables** เพิ่ม
   - `DATA_DIR` = `/data`
5. ไปที่แท็บ **Settings → Volumes → Add Volume** ตั้ง Mount Path = `/data`
6. ไปที่ **Settings → Networking → Generate Domain** จะได้ลิงก์เช่น `https://cat-food-order.up.railway.app`
7. เสร็จ! เปิดลิงก์ = หน้าสั่ง, ลิงก์ + `/admin.html` = หลังบ้าน

### ตัวเลือก B: Render (ฟรีจริง แต่ข้อมูลอาจรีเซ็ต)

Render มีแพ็กเกจฟรี แต่พื้นที่เก็บไฟล์เป็นแบบชั่วคราว — ถ้าเซิร์ฟเวอร์พักตัว (ไม่มีคนใช้ ~15 นาที) หรือ deploy ใหม่ **ข้อมูลออเดอร์จะหาย** เหมาะสำหรับลองเล่น หรือใช้แบบเคลียร์ออเดอร์ทุกวันอยู่แล้ว

1. เข้า https://render.com → Login ด้วย GitHub
2. กด **New → Web Service** → เลือก repo `cat-food-order`
3. ตั้งค่า:
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Instance Type: **Free**
4. กด **Create Web Service** รอ build เสร็จจะได้ลิงก์ `https://xxx.onrender.com`
5. เปิดลิงก์ = หน้าสั่ง, + `/admin.html` = หลังบ้าน

> อยากให้ Render เก็บข้อมูลถาวรแบบฟรี ต้องต่อฐานข้อมูลภายนอกเพิ่ม (เช่น MongoDB Atlas) — ถ้าต้องการแนวนี้บอกได้ เดี๋ยวปรับโค้ดให้

## การใช้งานหลัง deploy
- **แชร์ลิงก์หน้าสั่ง** (ลิงก์หลัก) ให้ลูกค้า/เพื่อน — เปิดในมือถือได้เลย กดสั่งอย่างเดียว
- **เก็บลิงก์หลังบ้าน** (`/admin.html`) ไว้ใช้เอง — เพิ่มร้าน เมนู รูป จัดการออเดอร์ ดูสรุป
- ข้อมูลซิงก์อัตโนมัติทุก 5 วินาที ทุกเครื่องเห็นตรงกัน
